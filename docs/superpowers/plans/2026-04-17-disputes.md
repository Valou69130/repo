# Disputes Implementation Plan

**Goal:** Ship the full dispute flow on top of the margin-call event store ‚Äî open, iterate, resolve, withdraw, escalate ‚Äî all hash-chained and permission-gated.

**Architecture:** New `src/routes/disputes.js` alongside existing `marginCalls.js`. Every state-advancing action funnels through `appendEvent` so the hash chain and `current_state` cache stay in sync. Dispute rows in `disputes` mirror the latest state; the authoritative history is the event log.

**Tech Stack:** Express 4, better-sqlite3, node:test + supertest (in-memory DB per test via `buildApp`).
